### First cycle (pre-fix)

All 4 specs failed AST validation with `invalid syntax`. Root cause: Grok wraps
codegen output in ``` ```python ... ``` ``` markdown fences. Committed a
one-function fix (`_strip_code_fence` in `llm_client.py`) that unwraps the fence
on the live-codegen path. Mock cassettes already contained raw source so were
unaffected.

### Second cycle (post-fix)

4 specs generated, pipeline behaviour:

| Spec | Stage | Status | Reason |
|---|---|---|---|
| `momentum_velocity` | validation | failed | `params['_prev_mid'] = ...` subscript assign â€” correctly blocked |
| `mean_reversion_microprice` | persisted | passed | n_trades=1 (dry-run synthetic) |
| `book_imbalance_pressure` | sandbox | failed | `KeyError` at import (`snapshot['best_bid_depth']`) |
| `volatility_breakout_spread` | validation | failed | `params['_prev_mid'] = ...` subscript assign |

### Observations

- **Grok wants to mutate `params` for state.** Two of four specs tried
  `params['_prev_mid'] = ...`. The AST whitelist correctly rejects, but the
  ideation prompt should probably spell out that strategies are pure functions
  with no persistent state across ticks.
- **Grok assumes dict keys exist.** `book_imbalance_pressure` referenced
  `snapshot['best_bid_depth']` directly without a `.get()` guard. Sandbox
  caught it cleanly.
- **The one passing spec** (`mean_reversion_microprice`) computed microprice
  from `snapshot['bids']`/`snapshot['asks']`, which matches the sample snapshot
  shape. It's a real-looking strategy.

### Prompt tweaks for next iteration

Add to `prompts/codegen_system.md`:

1. `compute_signal` must be a **pure function** â€” no mutation of `params` or
   `snapshot`, no module-level state. State that persists across ticks is not
   supported in 6c.
2. Defensively `.get()` every snapshot key you read; assume only `bids`,
   `asks`, `timestamp` are always present.
